Breadth First Search

Breadth First search is known as an uninformed search because it does not use any information about how far the robot has traveled or how far the robot is from the goal. BFS begins at the starting position of the robot (root node) and begins looking for the goal by expanding all of the successors of the root node. If those successors are not the goal, then BFS expands each of those nodes, and this loop continues until the goal node is reached. During the implementation of the algorithm we keep track of the nodes by maintaining a queue.

Depth First Search

Depth first search is the complement to breadth first search. It is also an uninformed search method. Starting from the root node it expands all the successors of the start node same as BFS. However after the first node, DFS always expands the last successor added.If DFS gets to a point where it can not find any more successors for the node it is currently expanding, it will then move to the newest discovered node that hasn’t been expanded. The stack data structure is used during implementation to maintain the Last In First Out order for nodes.

A-star Search

A* is like Dijkstra’s Algorithm in that it can be used to find a shortest path. A* is like Greedy Best-First-Search in that it can use a heuristic to guide itself. It combines the pieces of information that Dijkstra’s Algorithm uses (favoring vertices that are close to the starting point) and information that Greedy Best-First-Search uses (favoring vertices that are close to the goal). The heuristic followed by A* balances between the exact cost of the node from the start and the estimated cost of the node to the goal to optimize the final path.

Greedy Best First Search

The Greedy Best-First-Search algorithm works by visiting vertices in the graph starting with the object’s starting point following some estimate (called a heuristic) of how far from the goal any vertex is. Instead of selecting the vertex closest to the starting point, it selects the vertex closest to the goal. Greedy Best-First-Search is not guaranteed to find a shortest path.

Results

We observe that A* and BFS obtain the optimal path in all map scenarios. While, both A* and BFS have very high space complexity, A-star is more streamlined due to its heuristic. DFS has low space complexity and computation cost but it tends to result in suboptimal pathing. Greedy best first guided by its goal oriented heuristic tends to be fastest when goal is present in direct line of sight of the start but ends up being suboptimal otherwise.
